# PROBLEMS

Real-world problems submitted by the community. Real problems are the
fuel of this whole experiment.

## Submit a problem

Open a [Problem issue](../../../issues/new?template=problem.md) — that's
Mission 1, and the single most valuable first contribution.

When a problem gains traction, it gets registered here as a file with a
permanent ID.

## Problem record format

Create `PROB-NNN-short-title.md`:

```markdown
# PROB-NNN: [Title]

**ID:** PROB-NNN
**Description:** [What's wrong, for whom, and what "better" looks like]
**Source:** [Who submitted it — with consent if on someone's behalf — and link to the original issue]
**Status:** submitted | researched | in-experiment | in-venture | solved | archived
**Potential Impact:** [Who benefits and how much, honestly estimated]
```

## Rules

- Problems must be real: a specific person or group whose situation
  would actually improve if it were solved.
- Submitting someone else's problem requires their explicit consent.
- No problem is too small. Small and real beats big and vague.
