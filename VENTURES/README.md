# VENTURES

Problems that became ventures. When a problem's solution outgrows an
experiment and becomes something ongoing — a service, a product, a
recurring practice — its complete journey gets documented here.

## What "complete journey" means

Each venture gets a folder `VENTURES/<name>/` containing at minimum a
`README.md` that documents:

1. **Origin** — the problem it came from (link the `PROB-NNN` record)
2. **The experiment(s)** — what was tested and learned along the way
   (link the `EXPERIMENTS/` folders)
3. **The turning point** — what made this worth continuing beyond the
   experiment, in the words of the people it serves
4. **Who runs it now** — the humans accountable for it
5. **Current status** — honest, updated, including struggles
6. **What stays open source** — discoveries and patterns remain public
   even if the venture develops a life of its own

## Why we document the journey

Because the journey is the discovery. A venture that hides how it got
there teaches the community nothing. The path from problem → experiment
→ venture is exactly the knowledge this laboratory exists to produce.
