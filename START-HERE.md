# START HERE

Five minutes. That's all this page needs.
By the end, you'll know exactly what to do first.

---

## Minute 1 — What is this?

365EXPT is an open experiment testing one question:

> **Can AI create more meaningful work than it replaces?**

We test it by connecting real problems with people who want to solve
them, using AI as an assistant. Everything we learn is published here —
including failures.

## Minute 2 — Who decides things?

Humans. Always. AI (including [Divina](DIVINA/), our AI steward-in-
specification) helps organize, summarize, and match — it never governs
and never decides. The rules live in [BLUEPRINT/](BLUEPRINT/).

## Minute 3 — What counts as contributing?

Almost anything, and most of it needs no code:

- Submitting a real problem you've seen
- Improving documentation (this page included)
- Researching what already exists
- Designing better experiments
- Interviewing people affected by AI
- Translating documents
- Building integrations or improving Divina (this one is code)

## Minute 4 — Pick your first mission

| # | Mission | Code needed? | How |
|---|---------|--------------|-----|
| 1 | **Submit a problem** — a real one you've seen in the world | No | Open a [Problem issue](../../issues/new?template=problem.md) |
| 2 | **Improve documentation** — clarify anything confusing | No | Edit any `.md` file, open a PR |
| 3 | **Research an existing solution** — what already solves a listed problem? | No | Comment findings on a Problem issue |
| 4 | **Design a better experiment** — improve a method in [EXPERIMENTS/](EXPERIMENTS/) | No | Open an [Experiment issue](../../issues/new?template=experiment.md) |
| 5 | **Improve Divina** — refine her spec, prompt, or roadmap | Some | PR into [DIVINA/](DIVINA/) |
| 6 | **Build an integration** — Telegram, Discord, WhatsApp, MCP… | Yes | Start in [INTEGRATIONS/](INTEGRATIONS/) |
| 7 | **Interview someone affected by AI** — publish what you heard | No | Add a write-up to [DISCOVERIES/](DISCOVERIES/) via PR |
| 8 | **Translate documentation** — bring this to your language | No | PR a translated file |

Can't decide? **Mission 1 or Mission 2.** Both take under 30 minutes.

## Minute 5 — Make your first contribution

1. Choose a mission above.
2. For issues: click the link, fill in the template, submit.
3. For PRs: fork → edit → open a pull request. GitHub's web editor is
   enough — no local setup required for documentation.
4. A maintainer (and eventually Divina) will welcome you and respond.

That's it. You're a contributor.

---

## After your first contribution

- Your work becomes part of the public record — experiments you touch
  cite you.
- Check [DISCOVERIES/](DISCOVERIES/) to see what the community has
  learned so far.
- Join [Discussions](../../discussions) — ideas evolve there before
  they become implementations.
- Come back. The experiment compounds with every return visit.

## Need resources?

Some research needs infrastructure — hosting, API keys, compute time.
If your contribution needs something you can't access, email
**andrea@jamot.pro** with what you're working on and what you need.
We distribute resources from partners like ElevenLabs and Render to
support community research. See [CONTRIBUTING.md](CONTRIBUTING.md)
for details.

**Questions?** Open a [Discussion](../../discussions) — asking a good
question is itself a contribution.
